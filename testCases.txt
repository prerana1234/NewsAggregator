=============================
NEWS AGGREGATOR - TEST CASES
=============================

-----------------------------------
1. FUNCTIONAL TEST CASES
-----------------------------------

TC01 - Verify keyword-based search
Steps:
1. Enter a keyword (e.g., "apple")
2. Click Search
Expected:
- Aggregated news articles from Guardian and NYTimes are displayed
- Duplicates are removed

TC02 - Verify pagination with default values
Steps:
1. Leave Page No and Page Size empty
2. Perform search
Expected:
- Default pagination values are applied (Page No = 1, Page Size = 10)

TC03 - Verify pagination navigation
Steps:
1. Perform search
2. Click “Next Page” or “Previous Page” buttons
Expected:
- Correct page of articles is displayed
- Meta info updates accordingly

TC04 - Verify result refresh on parameter change
Steps:
1. Perform search for "apple"
2. Change keyword to "tesla" and search again
Expected:
- Results update with new keyword
- Meta info changes to reflect new search

TC05 - Verify city parameter (optional)
Steps:
1. Enter city “New York” with keyword “apple”
2. Perform search
Expected:
- Results are displayed (no error even if city filter is unused)

TC06 - Verify offline mode toggle
Steps:
1. Check “Offline” toggle
2. Perform search
Expected:
- Data is retrieved from local/offline cache
- No API calls to external services

TC07 - Verify API unavailability handling
Steps:
1. Disable or mock Guardian/NYT APIs
2. Perform search
Expected:
- Service still returns relevant cached or fallback results
- No crash or HTTP 500 errors

TC08 - Verify duplicate elimination
Steps:
1. Search for “apple”
2. Observe result list
Expected:
- Duplicate URLs or titles are not repeated

TC09 - Verify meta information section
Steps:
1. Perform a valid search
Expected:
- Displays keyword, total pages, page retrieved, prev/next page, time taken

TC10 - Verify time taken field
Steps:
1. Perform multiple searches
Expected:
- “Time Taken (ms)” shows realistic values (< 5000ms)

-----------------------------------
2. API TEST CASES (/api/news)
-----------------------------------

API01 - Valid keyword
Request: /api/news?keyword=apple&pageNo=1&pageSize=10
Expected: HTTP 200 OK, JSON array of articles

API02 - Empty keyword
Request: /api/news?keyword=&pageNo=1&pageSize=10
Expected: Empty results or validation message

API03 - Invalid page number
Request: /api/news?keyword=apple&pageNo=-1
Expected: Defaults to page 1 or error message

API04 - Missing parameters
Request: /api/news
Expected: Defaults applied for pagination, keyword handled gracefully

API05 - Verify structure of response
Expected JSON Fields:
keyword, city, totalPages, pageNo, prevPage, nextPage, timeTakenMillis, articles[]

API06 - Verify Guardian + NYT data merge
Expected:
- Combined response with unique articles from both APIs

API07 - Verify response under API failure
Steps:
- Simulate Guardian API down
Expected:
- NYT results still appear
- No 500 error

API08 - Performance test
Steps:
- Trigger multiple requests
Expected:
- Response within 3-5 seconds
